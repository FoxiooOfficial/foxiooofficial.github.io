Line Objects

29 November 2025

Line objects are defined as those objects that have a start and end point, and a 2D line is drawn on the screen through these points.

These objects mainly use 2 separate Sprite objects for points A and B.



Each point has its order, starting from point A to point B.

The object also has four behaviors: Set Points and Other Variables, Set Position, Rotation & Scale, Set Visibility, and Set Texture. The first behavior retrieves the position of points A and B as well as their visibility. The second behavior uses the obtained values to set the object on the screen, sets its scale where it should start and end, and calculates the angle at which the line should be drawn. The third behavior sets the visibility of the line, and the fourth sets the line's texture to 1 (You can modify this if you want the line to change texture).

Variables in Line Objects

This object has several variables in the Alterable Values section.

Variable Default Value Description Date Added
_PointX_A (Automatically) The variable that stores the X value of point A in the polygon shader. R1.0.1_10.05.25
_PointY_A (Automatically) The variable that stores the Y value of point A in the polygon shader. R1.0.1_10.05.25
_PointX_B (Automatically) The variable that stores the X value of point B in the polygon shader. R1.0.1_10.05.25
_PointY_B (Automatically) The variable that stores the Y value of point B in the polygon shader. R1.0.1_10.05.25
_Thickness 2 The variable that defines the thickness of the line drawn on the screen. R1.0.1_10.05.25
_TextureSizeX 1 Declare here the size of the line texture you are using on the X-axis. R1.0.1_10.05.25
_LayerOffset 0 Similar to Sprite, it shifts the object in layers. A positive value means the object will be above other objects; a negative value is the opposite. R1.0.1_10.05.25
_Layer (Automatically) The variable that stores which layer the polygon object is on. By default, it is the average of points A and B. R1.0.1_10.05.25

This object has several variables in the Flags section.

Variable Default Value Description Date Added
_Active (Automatically) This variable determines whether the polygon is active. If not, it sets the correct position on the screen and the object's scale before changing the value to 1. R1.0.1_10.05.25
_Visibility_A (Automatically) This variable determines if point A is visible. R1.0.1_10.05.25
_Visibility_B (Automatically) This variable determines if point B is visible. R1.0.1_10.05.25
_IgnoreShowing 0 This variable determines whether the polygon should ignore the visibility settings of points A, B, C, D and be invisible always or when needed. R1.0.1_10.05.25
_QualityScale 0 This variable determines whether the line should be drawn in a higher quality at the expense of performance. Setting it to 0 uses Nearest Neighbor filtering (pixelated), and setting it to 1 uses **Anti-Aliasing** filtering. R1.0.1_10.05.25
