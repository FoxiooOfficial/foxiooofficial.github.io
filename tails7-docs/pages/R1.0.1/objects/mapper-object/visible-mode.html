Tails7 Mapper Object - Visible Mode

29 November 2025

Visible Mode in the Tails7 Mapper Object is a behavior responsible for determining when an object from the 3D.Objects Group should be visible and when it shouldn't. Specifically, it sets the _Visible flag and, depending on the mode, can actually hide/show the object.

Different modes in Visible Mode

Visible Mode has 6 different modes that every object from the 3D.Objects Group can use to behave individually.  The string variable _VisibleMode is assigned a value that defines the given mode.

Value Description Date Added
"DEFAULT" This mode determines when the object should be visible or not and is mainly intended for **sprites**. Note that it takes the camera's direction into account and does not show objects that are behind the camera. The object's visibility range is 200° (<=100° / 260°>=).
(Also automatically sets the object's visibility) 		R1.0.1_10.05.25
"DEFAULTX" This mode determines when the object should be visible or not and is mainly intended for **sprites**. Note that it takes the camera's direction into account and does not show objects that are behind the camera. The object's visibility range is 200° (<=100° / 260°>=). R1.0.1_10.05.25
"CLOSEFADE" A mode similar to "DEFAULT", but this one is intended for **polygon points**, and the object's visibility range is 180° (<=90° / 270°>=).
(Also automatically sets the object's visibility) 		R1.0.1_10.05.25
"CLOSEFADEX" A mode similar to "DEFAULT", but this one is intended for **polygon points**, and the object's visibility range is 180° (<=90° / 270°>=). R1.0.1_10.05.25

Note

Setting a value other than those on the list will cause this behavior to be ignored.
