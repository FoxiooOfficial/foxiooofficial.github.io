Tails7 Mapper Object - Mapping Mode

29 November 2025

Mapping Mode in the Tails7 Mapper Object is one of the most important behaviors of the object, as it calculates most of how the object should be mapped.

Initially, in this behavior, the closest Tails 7 Render Ground relative to the Tails7 Center Point is always selected. This is intended to identify the main object that will be used for calculations. In this event, we set the most important variables on the Tails7 Cache object that will be used for Mapping Mode calculations.

Next, this behavior obtains the _DistanceX and _DistanceY variables of objects from the 3D.Objects Group, which were calculated by Distance Mode, and converts them to the final variables (_FinalX and _FinalY) for a position modified by perspective, similar to a shader.

Different modes in Mapping Mode

Mapping Mode has its own 4 modes that every object from the 3D.Objects Group can use to behave individually.  In the string variable _MappingMode, we assign a value that defines the given mode.

Value Description Date Added
"R1.0.1" The original version of the mode that converts the initial position to fit the perspective in the shader. This version does not use all camera rotations! R1.0.1_10.05.25
"R1.0.1_SKY" An additional mode that works similarly to "R1.0.1" but does not use the Z position for camera calculation, ideal for background objects such as clouds, stars, sun, mountains, and similar elements. This version does not use all camera rotations! R1.0.1_10.05.25
"R1.0.2" Currently the main mode that converts the initial position to fit the perspective in the shader.

This is a fix for the "R1.0.1" mode that takes into account the camera rotation on every axis. 		R1.0.2_18.05.25
"R1.0.2_SKY" An additional mode that works similarly to "R1.0.2" but does not use the Z position for camera calculation, ideal for background objects such as clouds, stars, sun, mountains, and similar elements.

This is a fix for the "R1.0.1_SKY" mode that takes into account the camera rotation on every axis. 		R1.0.2_29.11.25

Note

Setting a value other than those on the list will cause this behavior to be ignored.
