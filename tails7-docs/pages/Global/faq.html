FAQ

30 November 2025

Q: Why do polygons drawn by the Tails7 engine have graphical artifacts (e.g., edge distortions) and how can they be eliminated?

A: This problem is caused by the fact that when one of the points is too close to the camera and at a specific angle, its actual position is very large and stretches the polygon so much that artifacts appear. A temporary solution might be to use "1/2", "1/3" - meaning for one polygon, actually use two or three to create one wall, but this may increase the number of objects. I'm trying to fix polygons and it's possible that one of the future updates will resolve this.

Q: Are there differences in graphics APIs (e.g., Direct3D9 vs Direct3D11)?

A: There are differences. D3D11 has better optimization, while D3D9 is only useful in cases where D3D11 is not supported by the hardware.

Q: How can Tails7 be extended with custom mapping and scaling modes, and what are the implementation requirements?

All mappings are in the Tails7 Mapper Object, specifically in the behaviors. If you want to add a new scaling mode, copy the default one and paste it so it's in the same Sub-Event. It's important that the event doesn't contain the same name as other modes in that event.

Q: Is it possible to modify or extend the Tails7 engine's source code (e.g., edit shaders or add features)?

A: Tails7 is under the MIT license, which means you can share/modify it as you wish, but you must credit the authors and people mentioned in the acknowledgments.

Q: How long will the engine receive technical support?

A: This engine will be supported as long as I'm working on my game and other projects.
